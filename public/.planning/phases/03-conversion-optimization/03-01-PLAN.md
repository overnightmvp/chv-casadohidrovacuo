    **FIX 1: Add sticky trust column CSS**

    Find the existing `.quote-layout` section near the bottom of the file (appended in Phase 02). Find the existing `@media (max-width: 768px)` block for `.quote-layout`. After that block, ADD this new media query:

    ```css
    @media (min-width: 769px) {
      .quote-layout-trust {
        position: sticky;
        top: 80px;
        align-self: start;
      }
    }
    ```

    Note: `align-self: start` is needed because CSS Grid stretch would prevent sticky from working if the column stretches to fill the grid row.

    **FIX 2: Audit and fix .step-card CSS for fade-up interference**

    Search the file for any `.step-card` rule that contains `opacity` or `transform`. Likely the `.step-card` base rule has `opacity: 1` or there's a `.step-card-header`, `.step-card-body`, etc. that resets the transform.

    The fade-up pattern requires:
    - `.fade-up` starts: `opacity: 0; transform: translateY(24px);`
    - `.fade-up.visible` shows: `opacity: 1; transform: translateY(0);`

    If ANY `.step-card` rule has `opacity: 1` or `transform: none` or `transform: translateY(0)`, it will override `.fade-up` because `.step-card.fade-up` has higher specificity from `.step-card` than from `.fade-up` alone.

    **What to change:**
    - If `.step-card { opacity: 1; }` exists → REMOVE the `opacity: 1` line (keep other properties)
    - If `.step-card { transform: none; }` or similar → REMOVE it
    - If `.step-card:hover` has opacity change → keep it (hover is fine)
    - Do NOT remove any other step-card styles (border, padding, background, etc.)
    - Do NOT add any new step-card rules — only remove overrides

    Also check `.step-card-header`, `.step-icon`, `.step-number` for similar opacity/transform overrides and remove if found.
  </action>
